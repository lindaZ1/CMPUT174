User Documentation


General Information

Welcome to Ocean Observation System. This is a web-based database application system.

Ocean Observation System (OOS) is a computer information system that stores and processes different types of data for an ocean observatory. These data are produced by a wide range of different sensors, placed at different locations. In this system, users are able to upload, search, and download data provided by the sensors.

Sign up for a new account and explore a new field now!


Installation Guideline

  1. Download package and extract files to the path ~/catalina/webapps/Ocean

  2. Open terminal and execute commands in order:

    1. cd ~/catalina/webapps/Ocean

    2. javac WEB-INF/classes/*.java

    3. sqlplus ( enter < username>, < password> to start your sql)

    4. @setup.sql

    5. @populate.sql

    6. starttomcat

  3. Open a brower, enter http://u***.cs.ualberta.ca:<your-port-number>/Ocean/login.html



User Manual

  1. Login Module

Users: All Users

Function: This module enables all users to log in and log out of the Ocean Observation System. Users can also modify their personal information and password.

How to use:

Input your username and password. If you don't have one, please contact an administrator to create an account for you. After you log in, you will enter the home page of your role automatically.

You can manage your account by choosing the “My Account” option (which will appear on almost all pages), and you can see your detailed information.

You can modify your personal information by choosing the “Modify Personal Information” option. You may update all information except Person ID (which can only be edited by administrators). After editing, you will see your new information by clicking “save”.

You can modify your password by clicking “Change Password”. From here you have to enter your new password twice and click “Save”.

You can also go to the homepage of your role by clicking “Go to HomePage”.

If you want to log out, Please click “Log Out” button.

  1. Sensor management Module

Users: Only administrators

Function: This module enables administrators to create and remove sensors.

How to use:

Go to administrator page and choose the "edit sensor table" option. The system will then generate a sensor table that stores all information about sensors and prompts you to manage sensors.

To create a new sensor, you need to enter the correct sensor id, location, type and description of the sensor and choose the “create” option. Particularly, the sensor id you entered should not be in use, and the type you entered should be “a” or “i” or “s”, otherwise, there will be an error message.

To remove a sensor, you should enter the sensor id of the sensor you want to remove and choose the “remove” option. Also make sure that the sensor you want to remove is in the sensor table. Once you remove a sensor, all data stored on this sensor will be removed as well.



  1. User management Module

Users: Only administrators

Function: This module enables administrators to create, remove and update users.

How to use:

Go to administrator page and choose the "edit user table" option. The system will generate a persons table that stores all information about users and prompts you to manage users.

To create a new user, you need to enter the correct person id, first name, last name, address, email and phone number of the user, and choose the “create” option. Particularly, the person id and email you entered cannot be in use, otherwise, there will be an error message.

To remove a user, you should enter the person id of the user that you want to remove and choose the “remove” option. Also make sure that the user you want to remove is in the persons table. Once you remove a user, all data of this user will be removed as well.



  1. Subscribe Module

Users: Only scientists

Function: Enables scientists to subscribe and remove sensors from subscriptions

How to use: Go to scientist page and choose “Subscriptions” option. The system will then generate a table of all sensors with its sensor information and a table of sensors that the scientist is currently subscribed to.

To subscribe to a sensor, enter the sensor id and choose “Add”. This sensor id must exist in the all sensors table and cannot exist in the currently subscribed table.

To remove subscription to a sensor, enter the sensor id and choose “remove” option. This id must exist in the all sensors table and in the currently subscribed table.



  1. Uploading Module

Users: Only data curators

Function: Enables data curators to upload sensor data to database

How to use: Go to data curator page and choose “Upload” option. The system will then prompt you to choose a data type to upload. The types are scalar, image, and audio.

To upload scalar data, select “scalar” option. The system will prompt you to upload a .csv file with the format “sensor_id,datetime,scalar_value”. Upon completion, a message lets you know if your file has been uploaded.

To upload an image, select “image” option. The system will prompt you to upload a .jpg file, the system then prompts for sensor id, date, and description of the image. Upon completion, a message lets you know if your file has been uploaded.

To upload an audio clip, select “audio” option. The system will prompt you to upload a .wav file, the system then prompts for sensor id, date, description, and length of the audio clip. Upon completion, a message lets you know if your file has been uploaded.



  1. Search Module

Users : Only scientists

Function:

This module enables scientists to search and download from sensors according to search conditions specified.

How to use:

Go to scientist page and press "search" option. The system will prompt you to enter the keywords of description, location of sensors, type of sensors and time period of data. You may leave location and type empty, but make sure that you correctly enter keywords and time

After submitting your search conditions, the system will generate a list of images, audio recordings and scalar data of sensors that matches the search condition. Also, you can only see the data of sensors you have subscribed to. If you want to save data, you can download full size image, audio file and scalar data in csv file by choosing the "download" option.



  1. Data Analysis Module

Users: Only scientists

Function: This module enables scientists to view the average, minimum, and maximum values of the selected sensor on five levels of time hierarchies.

How to use:

After you enter Scientist homepage, you can enter Data Analysis page by clicking “Data Analysis” button.

Now you can see all sensors information and your subscribed sensors. Since you can only choose the sensors you have subscribed, you can click “Go To Subscribe” button to edit your subscribed sensors. After that, you can select which sensor you want to analyze and the time hierarchies. Then click “Submit” button.

Then you can see the detailed information of your selected sensor and its average, minimum, and maximum values according to the time hierarchies.
